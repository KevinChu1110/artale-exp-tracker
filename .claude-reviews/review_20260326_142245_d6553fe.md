### 3. **潛在問題**

#### A. 平台相容性風險 (Critical)
- **問題**：`core/ocr.py` 中的 `capture_gold` 函數直接呼叫了 `screencapture` 指令。
- **後果**：這是 macOS 專有的命令。若使用者在 Windows 或 Linux 上執行，此功能將完全失效並可能拋出 `FileNotFoundError`。
- **Odoo 視角**：如同在 Odoo 中直接使用 OS 特定的 path separator 而非 `os.path.join`。

#### B. OCR 誤判風險
- **問題**：金幣的 Fallback 邏輯（尋找 7-12 位的數字）過於寬鬆。
- **後果**：在某些解析度下，若 EXP 剛好也是 7 位數以上且未被過濾掉，系統可能會將 EXP 誤認為金幣總量，導致數據暴增。

#### C. UI 重入問題 (Re-entrancy)
- **問題**：在 `_on_record_gold` 中使用了 `QApplication.processEvents()`。
- **後果**：這可能導致使用者在 OCR 進行時多次點擊按鈕，觸發多個執行緒。
- **改善**：應在點擊後立即 `self._gold_btn.setEnabled(False)`，待結果回傳後再恢復。
